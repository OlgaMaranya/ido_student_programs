# Модуль «Учёт дополнительных квалификаций ИДО»

## Описание
Модуль предназначен для регистрации фактов обучения студентов ИрГУПС по программам дополнительного образования в Институте дополнительного образования (ИДО), формирования отчётности и обеспечения сквозной аналитики по образовательной траектории студента.

## Установка

### 1. База данных

Выполните SQL-миграцию для создания необходимых таблиц:

```bash
mysql -u username -p database_name < db_migration_ido_module.sql
```

**Важно:** Миграция предполагает, что в БД уже существуют таблицы ЕИС:
- `persons` — физические лица
- `stud_cards` — карточки студентов
- `stud_groups` — учебные группы
- `faculties` — факультеты
- `users` — пользователи системы

### 2. Настройка веб-сервера

Разместите файлы проекта в директории веб-сервера:
- `backend/` — бэкенд (PHP)
- `frontend/pages/index.html` — точка входа интерфейса

Убедитесь, что PHP имеет доступ к сессиям и может подключаться к БД ЕИС.

### 3. Интеграция с ЕИС

Отредактируйте файл `backend/api/api.php`, указав правильный путь к подключению БД:

```php
require_once __DIR__ . '/../../includes/db_connect.php'; // Ваш путь к подключению ЕИС
```

Или создайте собственное подключение в начале файла.

### 4. Настройка прав доступа

Назначьте роли пользователям через существующий механизм ЕИС (таблицы `user_roles`, `access_operator`):
- `ido_admin` — администратор ИДО
- `ido_operator` — оператор ИДО
- `dekanat_read` — сотрудник деканата
- `analyst` — аналитик / руководство

## Ролевая модель

| Роль | Описание | Права доступа |
|------|----------|---------------|
| `ido_admin` | Администратор ИДО | Полный доступ к справочникам, редактирование всех записей, экспорт, просмотр аудита |
| `ido_operator` | Оператор ИДО | Ввод и редактирование записей студентов, просмотр активных программ, экспорт |
| `dekanat_read` | Сотрудник деканата | Только просмотр по своим факультетам/группам |
| `analyst` | Аналитик / руководство | Просмотр сводных отчётов, экспорт агрегированных данных |

## Таблицы базы данных

### ido_program_types
Справочник типов программ ДПО.

| Поле | Тип | Описание |
|------|-----|----------|
| id | int | Идентификатор типа |
| name | varchar(250) | Название типа программы |
| code | varchar(20) | Внутренний префикс/код |
| hours_min | smallint | Рекомендуемый мин. объём часов |
| active | int | 1-активен, 0-скрыт из интерфейса |
| created_at | datetime | Дата создания |

### ido_programs
Справочник конкретных программ/курсов ИДО.

| Поле | Тип | Описание |
|------|-----|----------|
| id | int | Идентификатор программы |
| type_id | int | Ссылка на ido_program_types.id |
| code | varchar(20) | Внутренний код программы |
| name | varchar(250) | Полное наименование программы |
| hours | smallint | Объём программы в часах |
| doc_template | varchar(100) | Код утверждённого шаблона документа |
| active | int | 1-активна, 0-архив/скрыта |
| created_at | datetime | Дата создания |
| updated_at | datetime | Дата обновления |

### ido_student_programs
Учёт прохождения программ студентами.

| Поле | Тип | Описание |
|------|-----|----------|
| id | int | Идентификатор записи |
| stud_id | int | Ссылка на stud_cards.id |
| person_id | int | Ссылка на persons.id (сквозная идентификация) |
| program_id | int | Ссылка на ido_programs.id |
| start_date | date | Дата зачисления на программу |
| end_date | date | Дата окончания/выдачи документа |
| status | tinyint | 1-записан, 2-в процессе, 3-завершил, 4-не завершил, 5-отчислен |
| doc_number | varchar(50) | Номер итогового документа |
| doc_date | date | Дата выдачи документа |
| comment | text | Примечание оператора/ИДО |
| created_by | int | user_id создавшего запись |
| created_at | datetime | Дата создания |
| updated_at | datetime | Дата обновления |

### ido_audit_log
Журнал изменений (аудит).

| Поле | Тип | Описание |
|------|-----|----------|
| id | int | Идентификатор записи лога |
| record_id | int | id из ido_student_programs |
| action | varchar(10) | INSERT, UPDATE, DELETE |
| user_id | int | Кто выполнил действие |
| old_data | json | Старые данные |
| new_data | json | Новые данные |
| event_time | datetime | Время события |

## API Endpoints

Все запросы к API выполняются через `backend/api/api.php` с параметром `action`.

### Справочники

| Действие | Метод | Параметры | Описание |
|----------|-------|-----------|----------|
| `program_types_list` | GET | `all=1` (опционально) | Список типов программ |
| `program_type_create` | POST | JSON: `{name, code, hours_min, active}` | Создать тип |
| `program_type_update` | POST | JSON: `{id, name, code, hours_min, active}` | Обновить тип |
| `program_type_delete` | GET | `id` | Мягкое удаление типа |
| `programs_list` | GET | `type_id`, `all=1` | Список программ |
| `program_create` | POST | JSON: `{type_id, code, name, hours, doc_template, active}` | Создать программу |
| `program_update` | POST | JSON: `{id, type_id, code, name, hours, doc_template, active}` | Обновить программу |
| `program_delete` | GET | `id` | Мягкое удаление программы |

### Программы студентов

| Действие | Метод | Параметры | Описание |
|----------|-------|-----------|----------|
| `student_programs_list` | GET | `person_id` | Программы студента |
| `student_program_create` | POST | JSON: `{stud_id, person_id, program_id, status, start_date, end_date, doc_number, doc_date, comment}` | Добавить запись |
| `student_program_update` | POST | JSON: `{id, ...поля для обновления}` | Обновить запись |
| `student_program_delete` | GET | `id` | Удалить запись |

### Поиск и отчёты

| Действие | Метод | Параметры | Описание |
|----------|-------|-----------|----------|
| `students_search` | GET | `q`, `nzk`, `group_id`, `faculty_id`, `course` | Поиск студентов |
| `report_data` | GET | `date_from`, `date_to`, `faculty_id`, `group_id`, `type_ids`, `statuses`, `has_document` | Данные отчёта |
| `report_export` | GET | `format=xlsx\|csv`, + фильтры отчёта | Экспорт отчёта |

### Аудит

| Действие | Метод | Параметры | Описание |
|----------|-------|-----------|----------|
| `audit_log` | GET | `user_id`, `date_from`, `date_to`, `action_filter` | Журнал изменений |

### Примеры запросов

**Поиск студентов:**
```javascript
fetch('../backend/api/api.php?action=students_search&q=Иванов')
  .then(r => r.json())
  .then(data => console.log(data));
```

**Добавление программы студенту:**
```javascript
fetch('../backend/api/api.php?action=student_program_create', {
  method: 'POST',
  headers: {'Content-Type': 'application/json'},
  body: JSON.stringify({
    stud_id: 123,
    person_id: 456,
    program_id: 7,
    status: 1,
    start_date: '2024-01-15'
  })
});
```
