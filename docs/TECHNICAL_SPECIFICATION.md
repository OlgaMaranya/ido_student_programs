# Техническое задание
## Модуль «Учёт дополнительных квалификаций ИДО»

**Версия документа:** 1.0  
**Заказчик:** Институт дополнительного образования (ИДО) ИрГУПС  
**Исполнитель:** Отдел разработки ЕИС

---

## 1. Общие положения

### 1.1. Наименование и основание для разработки
- **Наименование системы:** Модуль «Учёт дополнительных квалификаций ИДО»
- **Основание:** Потребность в централизованном учёте квалификаций, рабочих профессий и программ переподготовки

### 1.2. Назначение системы
- Регистрация фактов обучения студентов ИрГУПС по программам ДО
- Формирование отчётности по приобретённым квалификациям
- Обеспечение сквозной аналитики по образовательной траектории
- Интеграция с ЕИС без дублирования данных

### 1.3. Цели и задачи
| Цель | Задачи |
|------|--------|
| Централизованный учёт | Создание справочников, привязка студентов к программам |
| Отчётность | Фильтрация, экспорт в XLSX/CSV |
| Сквозная идентификация | Использование stud_cards.id + persons.id |
| Минимизация дублирования | Использование существующих таблиц ЕИС |
| Защита ПДн | Ролевая модель, ограничение отображения |

---

## 2. Требования к системе

### 2.1. Функциональные требования

#### Справочники
- **ido_program_types**: типы программ ДПО (CRUD только ido_admin, мягкое удаление)
- **ido_programs**: конкретные программы (CRUD только ido_admin, мягкое удаление)

#### Учёт студентов
- Поиск по ФИО, зачётке, группе, факультету
- Карточка студента с историей программ ИДО
- Форма добавления программы с автоподстановкой данных

#### Отчётность
- Фильтры: период, факультет, группа, тип программы, статус, наличие документа
- Экспорт: XLSX (с форматированием), CSV (UTF-8 с BOM)

### 2.2. Нефункциональные требования
- Время отклика поиска: ≤ 2 сек. (до 50 000 записей)
- Формирование отчёта: ≤ 10 сек. (до 5 000 записей)
- Аутентификация через ЕИС
- Логирование всех CRUD операций

---

## 3. Ролевая модель

| Роль | Права |
|------|-------|
| ido_admin | Полный доступ к справочникам, всем записям, экспорт, аудит |
| ido_operator | Ввод/редактирование записей студентов, просмотр активных программ |
| dekanat_read | Только просмотр по своим факультетам/группам |
| analyst | Просмотр сводных отчётов, экспорт агрегированных данных |

---

## 4. Модель данных

См. файл `db_migration_ido_module.sql` для DDL-скрипта создания таблиц.

### Связи с таблицами ЕИС
- `ido_student_programs.stud_id` → `stud_cards.id`
- `ido_student_programs.person_id` → `persons.id`
- `ido_programs.type_id` → `ido_program_types.id`
